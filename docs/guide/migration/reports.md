# Reports migration

> **Deprecation notice**: The built-in reporting feature of the Wazuh dashboard plugin has been **deprecated in Wazuh 5.x**. Report generation and management are now handled entirely by the **Reporting** plugin. The plugin-specific **Management > Reporting** section is no longer available.
>
> The only migration action available is preserving existing PDF files generated by the 4.x plugin before uninstalling it.

---

## How reports worked in 4.x

In Wazuh dashboard 4.x, the Wazuh plugin provided a built-in reporting feature. When a user generated a report, the plugin created a PDF file and stored it on the dashboard server filesystem. Reports were organized per-user in a subdirectory derived from a hash of the username.

The storage path is derived from the `path.data` setting in `opensearch_dashboards.yml` (default: `/usr/share/wazuh-dashboard/data`):

```
<path.data>/wazuh/downloads/reports/<hashed_username>/
```

On a standard installation this resolves to:

```
/usr/share/wazuh-dashboard/data/wazuh/downloads/reports/<hashed_username>/
```

Where `<hashed_username>` is a hash of the authenticated user's username. Each user's reports are stored in a separate subdirectory.

Users could view and download previously generated reports from the **☰ Menu > Management > Reporting** section of the dashboard.

### Custom report branding in 4.x

The 4.x plugin allowed customizing the PDF report appearance through `wazuh.yml`:

- `customization.logo.reports` — logo placed at the top left of every report page.
- `customization.reports.header` — text displayed in the report header.
- `customization.reports.footer` — text displayed in the report footer.

These settings are **not available in 5.x**. See the [Configuration migration](./configuration.md#customization-settings) guide for details.

---

## How reports work in 5.x

Report generation is now handled by the **Reporting** plugin, which is included with the Wazuh dashboard 5.x package. Reports are generated directly from dashboard views and stored in an index, not on the server filesystem.

> **Note**: The OpenSearch Dashboards Reporting plugin (`reportsDashboards`) is also bundled with Wazuh 4.14.x. The key difference in 5.x is that the Wazuh plugin's own built-in PDF generation (stored on the filesystem) has been removed entirely; the OSD Reporting plugin is now the sole reporting mechanism.

The Wazuh plugin no longer provides a dedicated reporting section. PDF and CSV reports are generated using the **Generate report** button available within individual dashboard views and can be accessed from **☰ Menu > Reporting** (provided by the OpenSearch Dashboards Reporting plugin, registered under the **All** navigation group).

---

## What can and cannot be migrated

| Item                                          | Migratable                                                                                                                                                               |
| --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| PDF files generated in 4.x                    | **No** — 5.x stores reports in an OpenSearch index; 4.x PDF files cannot be imported.                                                                                    |
| The ability to regenerate reports             | **Yes** — Migrate the underlying dashboards (see [Custom dashboards and visualizations](./dashboards.md)) and generate new reports from them using the Reporting plugin. |
| Custom report branding (logo, header, footer) | **No** — Not configurable in 5.x.                                                                                                                                        |

There is no automated path to import 4.x report files into the 5.x reporting system. Preserve 4.x PDF files as standalone documents if they are needed for compliance, auditing, or reference.

---

## Before uninstalling 4.x: preserve existing reports

Copy all existing PDF reports from the server filesystem before uninstalling the Wazuh dashboard 4.x package. Because reports are stored per user, you must copy the entire `downloads/reports/` directory to capture all users' reports.

### Copy reports from the filesystem

Run the following command **on the 4.x dashboard server**. Adjust the source path if `path.data` was customized in your `opensearch_dashboards.yml`:

```bash
sudo cp -r /usr/share/wazuh-dashboard/data/wazuh/downloads/reports/ \
   ~/wazuh-reports-backup-$(date +%Y%m%d)/
```

> **Note**: The `reports/` directory is owned by the `wazuh-dashboard` system user. Run the command with `sudo` or as a user with sufficient permissions to read the directory.

This copies all per-user subdirectories. Each subdirectory is named after a hash of the corresponding username. Store the backup in a location that is accessible independently of the Wazuh dashboard installation.

### Download reports through the UI (optional)

If you prefer to download reports individually before the backup:

1. Navigate to **☰ Menu > Management > Reporting**.
2. Download each report listed using the download action.

## Generating reports in 5.x

After completing the migration to Wazuh dashboard 5.x:

1. Ensure the **OpenSearch Dashboards Reporting** plugin is installed and enabled. The plugin is included with the Wazuh dashboard 5.x package.
2. Navigate to the dashboard view from which you want to generate a report.
3. Click **Generate report** in the toolbar.
4. The report is stored in the OpenSearch Dashboards Reporting index and is accessible from **☰ Menu > Reporting** (the Reporting app is registered under the **All** navigation group, not under Management).

> **Note**: To generate reports, the logged-in user must have the appropriate permissions in the `reports_read` and `reports_write` roles, or an equivalent role with access to the reporting index and API endpoints.

---

## Notes

- PDF files preserved from 4.x remain valid standalone documents and can be shared or archived independently of the Wazuh dashboard.
- The visual layout of 5.x reports may differ from 4.x reports because the underlying dashboard structure and rendering engine have changed.
- CSV exports from tables (governed by the `reports.csv.maxRows` Advanced Setting) continue to work in 5.x and do not require migration.
